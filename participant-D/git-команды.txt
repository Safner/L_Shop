Участник D — залить авторизацию и фронт на Git
================================================================================

Полный сценарий команды: ..\ГАЙД-3-УЧАСТНИКА.txt

Корень репозитория: L_Shop
Папка: participant-D\  (готовые файлы для копирования в проект)
Ветка для D (пример): auth  (или d-auth — как договоритесь)

1) Подтянуть main и при необходимости влить работу других веток (tovari, korzina), затем:

   git fetch origin
   git checkout main
   git pull origin main

2) Создать ветку и перенести файлы из participant-D в src/, public/, package.json
   (см. КАК_ПЕРЕНЕСТИ_В_ПРОЕКТ.txt).

   npm install
   npm run build

3) Ветка и коммит:

   git checkout -b auth
   # если ветка auth уже есть: git checkout auth

   git add src/ public/ package.json package-lock.json
   git status
   git commit -m "feat: регистрация, вход JWT, защита API (участник D)"
   git push -u origin auth

После npm install при необходимости снова: git add package-lock.json && git commit --amend
(или отдельный коммит с обновлением lock-файла).
